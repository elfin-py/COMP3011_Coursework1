\hypertarget{style-forecast-technical-report}{%
\section{Style Forecast --- Technical
Report}\label{style-forecast-technical-report}}

\hypertarget{overview}{%
\subsection{Overview}\label{overview}}

Style Forecast is a climate- and trend-aware outfit recommendation
system. It combines local weather forecasts, social/news trend signals,
and user feedback to produce outfit guidance and explainability. The
system offers an API-first architecture with a React client for
interaction and demonstration.

\hypertarget{technology-stack}{%
\subsection{Technology Stack}\label{technology-stack}}

\begin{itemize}
\tightlist
\item
  \textbf{Language}: TypeScript (backend + frontend)
\item
  \textbf{Backend}: NestJS + Prisma + PostgreSQL
\item
  \textbf{Frontend}: React + Vite
\item
  \textbf{Auth}: JWT (access + refresh tokens)
\item
  \textbf{Trends}: RSS ingestion + keyword/sentiment extraction,
  Pinterest image retrieval
\item
  \textbf{Weather}: Open-Meteo (live forecast + timezone resolution)
\end{itemize}

\hypertarget{rationale}{%
\subsubsection{Rationale}\label{rationale}}

\begin{itemize}
\tightlist
\item
  \textbf{TypeScript} provides shared language across client/server,
  reduces integration errors, and improves refactor safety.
\item
  \textbf{NestJS} offers structured modules, dependency injection, and
  straightforward testability.
\item
  \textbf{Prisma + PostgreSQL} provides strong relational modelling,
  migrations, and type-safe querying.
\item
  \textbf{React + Vite} yields fast dev iterations and a responsive UI
  for demonstration.
\item
  \textbf{Open-Meteo} is a free, documented source for live forecasts
  and timezone offsets.
\end{itemize}

\hypertarget{architecture}{%
\subsection{Architecture}\label{architecture}}

\begin{itemize}
\tightlist
\item
  \textbf{Client}: React app with recommendation flow, chat assistant,
  and saved outfits.
\item
  \textbf{API}: NestJS modules for auth, users, climate, trends,
  recommendations, outfits, feedback.
\item
  \textbf{Data}: PostgreSQL stores users, profiles, outfits, climate
  snapshots, trend signals, and feedback.
\end{itemize}

\hypertarget{data-model-summary}{%
\subsubsection{Data Model (summary)}\label{data-model-summary}}

\begin{itemize}
\tightlist
\item
  \texttt{User} + \texttt{Profile}: authentication + preferences + home
  location
\item
  \texttt{ClimateSnapshot}: weather at a time + location
\item
  \texttt{Outfit} + \texttt{OutfitUsage}: saved outfits and usage
  tracking
\item
  \texttt{SocialTrend}: trend tags with sentiment and volume
\item
  \texttt{Feedback}: user ratings for adaptive tuning
\end{itemize}

\hypertarget{core-features}{%
\subsection{Core Features}\label{core-features}}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Weather-aware recommendations} using live forecasts at
  user-selected location/time.
\item
  \textbf{Trend-aware scoring} based on recent media mentions and
  sentiment (last 60 days).
\item
  \textbf{Adaptive feedback} to bias future recommendations.
\item
  \textbf{Chat assistant} to refine outfit guidance with user intent and
  preferences.
\item
  \textbf{Saved outfits} for logged-in users and optional daily
  recommendation settings.
\end{enumerate}

\hypertarget{recommendation-logic}{%
\subsection{Recommendation Logic}\label{recommendation-logic}}

\begin{itemize}
\tightlist
\item
  Base score is a weighted blend of temperature match, precipitation
  risk, wind penalty, and trend boost.
\item
  Trend boost is sentiment-weighted using recent fashion media signals.
\item
  User feedback adds an adaptive boost.
\item
  Chat intent is treated as a soft bias (occasion, activity, avoid
  tags).
\end{itemize}

\hypertarget{weather-accuracy}{%
\subsection{Weather Accuracy}\label{weather-accuracy}}

\begin{itemize}
\tightlist
\item
  Uses Open-Meteo hourly forecasts with timezone resolution from
  geocoding.
\item
  The selected datetime is converted to the correct timezone before
  fetching.
\item
  If a live fetch fails, a clear error is returned (no stale fallback).
\end{itemize}

\hypertarget{trends-and-sentiment}{%
\subsection{Trends and Sentiment}\label{trends-and-sentiment}}

\begin{itemize}
\tightlist
\item
  RSS sources are polled to extract keyword mentions.
\item
  Sentiment is derived from positive/negative language cues in recent
  articles.
\item
  Only articles from the last 60 days influence \texttt{news-rss} trend
  signals.
\end{itemize}

\hypertarget{testing-approach}{%
\subsection{Testing Approach}\label{testing-approach}}

\begin{itemize}
\tightlist
\item
  \textbf{Manual}: API endpoints verified via curl/Swagger; UI flows
  validated in the browser.
\item
  \textbf{Automated}: Minimal unit coverage currently; key candidates
  are climate/timezone conversion and trend sentiment parsing.
\end{itemize}

\hypertarget{challenges-lessons}{%
\subsection{Challenges \& Lessons}\label{challenges-lessons}}

\begin{itemize}
\tightlist
\item
  \textbf{Timezone correctness} required server-side handling to avoid
  client-local time errors.
\item
  \textbf{Trend sourcing} needed reliable sentiment handling and recency
  windows to avoid stale results.
\item
  \textbf{Pinterest} image retrieval required proxying and
  de-duplication to avoid broken or irrelevant images.
\end{itemize}

\hypertarget{limitations}{%
\subsection{Limitations}\label{limitations}}

\begin{itemize}
\tightlist
\item
  Chat assistant depends on external AI availability; fallback is
  heuristic-based.
\item
  Trend sentiment is lexical and can be noisy.
\item
  Limited automated testing coverage.
\end{itemize}

\subsection{Future Work}\label{future-work}}

\hypertarget{generative-ai-declaration}{%
\subsection{Generative AI Declaration}\label{generative-ai-declaration}}

Generative AI was used to assist with small coding corrections and
debugging, summarising design choices, and document scaffolding. All
outputs were reviewed and edited to ensure correctness. AI-generated
suggestions were implemented only after validation against the system's
functional requirements and observed behaviour.
